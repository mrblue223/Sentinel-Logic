# 🔒 Security Policy

## Supported Queries

We actively maintain and review queries in this repository. If you find that a query produces incorrect results, generates excessive false positives, or has a logic flaw, please report it.

## Reporting a Problem

**Please do not open a public GitHub issue for security-sensitive reports.**

If you discover a query that could be weaponized, misused, or reveals sensitive detection logic that could help attackers evade detection, contact us privately:

- Open a [GitHub Security Advisory](../../security/advisories/new) (preferred)
- Or contact the maintainer directly via GitHub profile

We will respond within **72 hours** and work with you to assess and address the issue.

## Scope

This repository contains **defensive** KQL queries only. We do not accept offensive tooling, evasion techniques, or anything that primarily serves to help attackers bypass detections.

## Responsible Disclosure

We appreciate responsible disclosure and will acknowledge contributors who report valid issues in our release notes.
